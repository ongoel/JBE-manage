# JBE 매니저 시트 구조 설계서

## 1. 개요
이 문서는 JBE 매니저 시스템에서 사용하는 구글 스프레드시트의 구조를 정의합니다.

## 2. 공통 설정
- **기본 폰트**: 맑은 고딕 (Malgun Gothic)
- **헤더 행**: 1행 (굵게, 배경색 지정)
- **데이터 시작**: 2행부터

## 3. 시트별 상세 정의

### 3.1 등록부 (Registry)
회원 전체 명단을 관리하는 시트입니다.
- **시트명**: `등록부`

| 열 | 헤더명 | 설명 (Description) | 데이터 예시 | 비고 |
|:---:|:---|:---|:---|:---|
| A | 회원번호 | 고유 식별자 (ID) | `M001`, `M002` | 자동 생성 |
| B | 성명 | 회원 이름 | `홍길동` | 동명이인 가능 |
| C | 소속 | 소속 기관/팀 | `A팀`, `B팀` | |
| D | 등번호 | 유니폼 번호 | `10`, `7` | 중복 불가 |
| E | 주 포지션 | 선호하는 주 포지션 | `FW`, `MF` | 4대 포지션 중 택1 |
| F | 선호 포지션 | 2순위 포지션 | `DF` | 선택 사항 |
| G | 주발 | 주로 사용하는 발 | `R` (오른발), `L` (왼발) | |
| H | 상태 | 회원 상태 | `활동`, `휴면`, `탈퇴` | |
| I | 가입일 | 최초 등록일 | `2024-01-01` | `YYYY-MM-DD` |

### 3.2 출석부 (Attendance)
연도별 출석 현황을 관리하는 시트입니다.
- **시트명**: `출석부_{YYYY}` (예: `출석부_2026`)

| 열 | 헤더명 | 설명 | 비고 |
|:---:|:---|:---|:---|
| A | 회원번호 | `등록부`와 연동 | |
| B | 성명 | `등록부` A열 참조 (VLOOKUP) | |
| C | 등번호 | `등록부` D열 참조 | |
| D | 소속 | `등록부` C열 참조 | |
| E~ | {날짜} | 해당 날짜 출석 여부 (1/0) | 동적으로 열 추가 |
| 마지막 | 합계 | 연간 총 출석 횟수 | `=SUM(...)` |

### 3.3 등번호 현황 (Number Status)
사용 중인 등번호와 사용 가능한 등번호를 한눈에 파악합니다.
- **시트명**: `등번호 현황`

| 열 | 헤더명 | 설명 |
|:---:|:---|:---|
| A | 등번호 | 0~99번 |
| B | 사용자 | 사용 중인 회원명 (없으면 빈칸) |
| C | 상태 | `사용중`, `가능` |

## 4. 데이터 유효성 검사 (Validation)
- **포지션 열**: `GK`, `DF`, `MF`, `FW` 목록에서 선택
- **주발 열**: `R`, `L`, `Both` 목록에서 선택
- **상태 열**: `활동`, `휴면`, `탈퇴` 목록에서 선택
