### 추천모델은 2가지(추천, 차선책)로 제시해주세요.

* **[보안 및 백업 설정]**
    * 1. API key, DB key 등은 보안상 코드에 직접 입력하지 않고, GAS 프로젝트 설정에 스크립트 속성으로 관리합니다.
    * 2. 매일 1회(특정시점) 스프레드시트 전체를 '백업시트'에 백업(복구 버튼으로 복구 가능하게 설계)
    * 3. 매일 1회(특정시점) 롤백시점을 기록하여, 롤백버튼으로 롤백 시점의 스프레드시트를 복구 가능하게 설계
    * 4. 현재 대시보드 첫 화면에 회원정보(이름, 소속)가 표시되는데, 회원현황(전체 회원 수)만 표시하고, 개인정보는 표시하지 않도록 변경
    * 5. 별도의 관리자 메뉴를 만들어 관리자 및 접근권한을 가진 이용자 설정하여, 권한이 있는 이용자(간부: 회장, 감독, 수석총무, 기획총무, 총무, 코치, 홍보)만 접근 가능하도록 변경(아이디, 비밀번호 설정)
    * 6. 아이디 비밀번호의 보안을 위해 코드에 직접 입력하지 않고, GAS 프로젝트 설정에 스크립트 속성으로 관리합니다.(추가적으로 아이디, 비밀번호를 초기화 할 수 있는 기능 추가)
    * 7. 보안취약점 점검 및 개선점 제시

* **[기능변경 및 추가기능 설정]**
    * 1. 주 포지션 선택제한을 3개로 변경(기존 2개, 구글 폼 설정 변경 필요)
    * 2. 선호 포지션 선택제한을 삭제(제한 없이 입력 가능하도록 변경, 구글 폼 설정 변경 필요)
    * 3. 메인화면에는 앞서 언급한 '회원현황(전체 회원 수)'만 표시하고, 개인정보는 표시하지 않도록 변경
    * 4. 메인화면에 총 동호회 현황, 이달의 평균출석률(활동회원만으로 통계)
        - 활동회원 기준: 최근 3개월 이내 1회 이상 출석한 회원
        - 월별 평균출석률: 해당 월에 참석한 회원의 수를 해당 월의 활동회원 수로 나눈 값
        - 월별 평균출석률은 대시보드에 그래프로 표시(최근 3개월치, 1주단위로 꺽은선 그래프로 표시)
    * 5. 회원명단 출석부 순서을 다음과 같이 조정(직급에 따라), 수동으로 변경 가능하도록 버튼을 눌렀을 때만 새로 정렬되도록(버튼은 대쉬보드 관리자 메뉴에 구현)
        - 1. '명예회원'
        - 2. '회장'
        - 3. '감독'
        - 4. '수석총무'
        - 5. '기획총무'
        - 6. '총무'
        - 7. '코치'
        - 8. '홍보'
        - 9. 이 외의 회원은 이름을 기준으로 가나다 순으로 정렬
    * 6. 매 경기 후 참석회원들의 경기내용에 대한 피드백을 입력받아, 선수별 실력을 점수로 변환하여 이후 팀구성에 사용됩니다.(대쉬보드 관리자 메뉴에 구현)
        - 경기평가는 관리자 및 간부(회장, 감독, 수석총무, 기획총무, 총무, 코치, 홍보) 권한이 있는 이용자만 입력 가능
            - 초기 기능은 아이디 비밀번호를 만든 간부들만 입력 가능하도록 구현
            - 향후 일반회원도 간부들이 승인하면 입력 가능하도록 변경할 수 있도록 설계
        - 평가지표는 동호회 회원이 아마추어이므로 프로선수 기준이 아닌, 동호회 내에서 평가하는 기준으로 10점 만점으로 평가하며, 평가지표는 최대한 객관적으로 평가가능 하도록 설계
            - 공격력, 수비력, 패스능력, 팀워크, 체력, 정신력, 기술, 전술이해도를 평가
            - 각 항목별로 1~10점까지 평가
            - 선수를 선택하면 해당 선수의 각 지표를 육각형 그래프로 표시
            - 종합점수를 매기고, 전체 또는 포지션별(FW, MF, DF, GK) 종합점수 내림차순, 올림차순으로 정렬하는 기능 구현
                - 세부포지션: FW(ST, CF, SS, LW, RW), MF(CM, CDM, CAM, LM, RM), DF(CB, LB, RB, LWB, RWB), GK(GK)
        - 평가지표는 수동으로 추가, 삭제, 수정 가능하도록 설계(관리자 및 간부 권한만 가능)
    * 7. 네이버 밴드 포스팅 기능(포스팅 버튼을 관리자 대쉬보드로 구현하여, 권한있는 이용자만 포스팅 가능하도록 설계, 자동포스팅 하지 않도록)
        - 포스팅할 내용을 미리 작성해두고, 포스팅 버튼을 누르면 네이버 밴드에 포스팅
        - 구글 시트의 '경기일정' 시트의 내용을 기반으로 자동으로 포스팅 하는 기능 구현(실험단계에서는 실험용 밴드 생성하여, 혼자 테스트 진행)
            - 포스팅할 내용을 미리 작성해두고, 포스팅 버튼을 누르면 네이버 밴드에 포스팅
        - '경기일정' 시트에 필요한 정보(하단의 시간 및 날짜는 한국표준 시간 기준)
            - 자체경기
                - 경기날짜: (YYYY.MM.DD)(요일)
                - 집합시간: (HH:MM)
                - 경기시간: (HH:MM ~ HH:MM)
                - 장소: (장소명)
                - 내용
                - 비고
            - 외부경기
                - 경기날짜: (YYYY.MM.DD)(요일)
                - 집합시간: (HH:MM)
                - 경기시간: (HH:MM ~ HH:MM)
                - 장소: (장소명)
                - 상대팀: (상대팀명)
                - 내용
                - 비고
        - 포스팅한 글 아래부분에 투표 기능 활성화(밴드에 포스팅할 때 자동으로 포스팅한 내용 하단에 투표를 추가하여 활성화)
            - 경기참석여부 투표
                - 투표내용: 참석, 불참
                - 투표기간: 제한없음
                - 중복 불가능
                - 무기명투표 불가능
            - 식사 참석여부 투표
                - 투표내용: 참석, 불참
                - 투표기간: 제한없음
                - 중복 불가능
                - 무기명투표 불가능
        - 향후 '경기일정'을 미리 등록해둘 수 있도록 설계
            - 이미 지나간 경기의 일정은 포스팅 하지 않게 설계
            - 포스팅할 경기 일정을 선택할 수 있도록 설계(지나간 날짜는 선택 불가능)
    * 8. 밴드 참석 투표인원 수동으로 입력 가능한 '투표인원' 시트 생성
        - '등록부'와 '출석부' 시트의 내용을 기반으로 '이번주 참석인원' 시트를 생성
        - 참석인원의 포지션별 인원수를 바탕으로 부족한 포지션 표시(대시보드 관리자 메뉴에 해당 기능 구현)
    * 9. 팀밸런스 기능(추천 팀원 명단) 실제 구현(대시보드 관리자 메뉴에 해당 기능 구현)
        - '이번주 참석인원' 시트의 내용을 기반으로 팀을 자동으로 구성
        - 팀 구성(옵션으로 구성변경 가능하도록 설계)
            - A팀 VS B팀(기본설정): 참석회원 실력점수를 고려하여 구성
            - 기관 VS 학교: 이벤트 경기이므로 기관 소속과 학교 소속을 구분하여 기계적으로 구성(점수 고려하지 않음)
        - 팀 밸런스를 위해 포지션별 실력점수를 고려하여 팀 구성
        - 팀 구성은 포지션별로 인원수를 맞추어 구성
        - 추천 팀원 명단은 '추천팀원' 시트에 저장(경기일정별 구분, 수동으로 변경 가능하도록 설계)
